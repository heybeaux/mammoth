# Mammoth v0.1.0 MVP acceptance receipt

## Checkpoint criterion

One local deterministic evidence-first research program can run end to end,
survive interruption, and produce an inspectable dossier whose factual sentences
resolve through named policy assessments and exact locators to immutable evidence.

## Acceptance evidence

The CI job runs frozen-lockfile installation, formatting, lint, typecheck, build,
tests, and offline evaluation. `pnpm verify:mvp` composes the Phase 0, Phase 1,
Phase 2, M2, M3, and independent black-box MVP gates.

The independent acceptance verifier imports no Mammoth domain or compiler code
for its integrity decisions. It:

- launches the declared CLI in fresh OS processes;
- recomputes CAS byte digests and canonical receipt/artifact digests;
- traverses report sentence → claim → assessment → policy/version → evidence
  edge → exact locator → snapshot digest;
- proves unsupported and unresolved claims cannot render as supported facts;
- injects interruption at all five declared runtime stages, then checks
  idempotent convergence and exactly-one effect receipt;
- validates cancellation partial receipts and new-process artifact readability;
- confirms a deterministic +24-hour evidence revalidation schedule;
- checks governance audit sequence continuity; and
- rejects tampered receipts, traces, ledgers, charters, and symlink/path escapes.

Budget exhaustion, lease recovery, retry/dead-letter behavior, human-gate expiry,
and provider-idempotent crash recovery are proven by `pnpm verify:phase-2`, which
is a required sub-gate of `verify:mvp`.
