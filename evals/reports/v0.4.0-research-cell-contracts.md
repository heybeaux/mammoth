## Candidate scope and authority

P4 introduces deterministic, versioned contracts for research-cell plans,
positions, review assignments, reviews, dissent, synthesis, outcomes, receipts,
model profiles, immutable model-profile versions, model lineage, and correlation
assessment. Deterministic policy code admits or rejects proposals; model agreement
does not promote evidence or truth.

Postgres and CAS remain authoritative for product state and artifacts. Temporal
contracts carry stable identifiers and bounded orchestration metadata, then
reconstruct product state through inward-facing ports. Observatory projections
remain deterministic, read-only views over validated authoritative snapshots.

`pnpm verify:p4` first executes every manifest evaluator, then runs these eight
code-owned gates:

- `fixture-manifest` — executable adversarial fixtures;
- `domain-policy` — contracts, lineage, correlation, and admission;
- `persistence-contracts` — authoritative ports, digests, and reconstruction;
- `postgres-repositories` — migration, immutability, concurrency, fencing, and
  integrity;
- `native-postgres-restart` — real local Postgres install/restart/reconstruction;
- `workflow-carriage` — stable IDs, bounded carry, and state reconstruction;
- `observatory-projection` — read-only topology and fail-closed projection; and
- `operator-inspection` — read-only CLI inspection.

The verifier is non-recursive and requires every target package script to exist
and exit successfully. The feature branch adds `pnpm verify:p4` to the CI workflow,
but default-branch enforcement remains pending until the change is merged and the
main push run is observed green.

## Known limitations and P5 deferral

- P4 freezes contracts, policy, persistence, carriage, and projection behavior; it
  does not claim execution-time peer isolation.
- Blind review context, commit-before-reveal position isolation, sanitized reviewer
  inputs, and isolated multi-model fan-out remain P5 work.
- Managed Postgres, managed object storage, and managed Temporal deployment are not
  established by this checkpoint.
- The 3D Observatory UI remains deferred; P4 supplies only the read-only projection
  and operator-inspection contract.
- Model lineage describes correlation risk, not evidence authority. Unknown
  lineage is conservative and cannot be counted as independent support.
- Temporal history remains orchestration evidence, not a product-state or truth
  store.
